# How to Use: iptables-explain Skill

## Installation

### 1. Clone the skills library

```bash
git clone https://github.com/ranga-sampath/claude-skills-library
cd claude-skills-library/iptables-explain
```

### 2. Install the skill globally in Claude Code

Claude Code discovers skills from `~/.claude/skills/` — your global skills directory. Copying the files there registers the skill for every Claude Code session on this machine, regardless of which project directory you are working in.

```bash
mkdir -p ~/.claude/skills/iptables-explain
cp .claude/skills/iptables-explain/skill.md ~/.claude/skills/iptables-explain/
cp .claude/skills/iptables-explain/iptables_parser.py ~/.claude/skills/iptables-explain/
```

### 3. Verify the skill is available

Open Claude Code and type `/` — `iptables-explain` should appear in the autocomplete list.

---

## Capturing the Snapshot

```bash
# IPv4 rules
iptables-save > current.txt

# IPv4 rules with packet/byte counters
iptables-save -c > current-with-counters.txt

# IPv6 rules
ip6tables-save > current-ip6.txt
```

The file must be the output of `iptables-save`. Raw `iptables -L` output is not supported.

---

## Usage

```
/iptables-explain <snapshot.txt>
```

### Examples

```
# Analyse the current host's IPv4 rules
/iptables-explain /tmp/current.txt

# Analyse a snapshot captured from a remote host
/iptables-explain ~/snapshots/prod-web-01-20260315.txt

# Analyse IPv6 rules
/iptables-explain /tmp/current-ip6.txt
```

Claude will:
1. Check the file exists and the parser script is installed
2. Run `iptables_parser.py` to produce structured JSON
3. Analyse the JSON and return the full explanation inline

## What the Report Covers

The report follows this structure:

| Section | Content |
|---|---|
| **Framework** | `iptables-nft` or `iptables-legacy`, with a brief explanation |
| **Default Policies** | Table of chain → policy; DROP policies highlighted |
| **Rules** | Chain-by-chain plain-English explanation |
| **Security Posture Summary** | 2-4 sentence verdict |
| **Notable Findings** | Anything that warrants attention |

## Understanding the Output

### Framework

The header line of every `iptables-save` file identifies the variant:
- `iptables-save v1.8.x (nf_tables)` → **iptables-nft** (Ubuntu 20.04+, Debian 11+)
- `iptables-save v1.8.x` (no suffix) → **iptables-legacy**

Both have identical syntax. The distinction matters if native `nft` rules also exist on the host.

### Custom Chains

The skill identifies known chain patterns:

| Pattern | What it is |
|---|---|
| `f2b-*` | fail2ban ban chain — lists active bans (IPs before RETURN) |
| `DOCKER*` | Docker networking — isolates container traffic |
| `ufw-*` | UFW (Uncomplicated Firewall) managed rules |
| `KUBE-*` | Kubernetes networking |

### Counters

If the snapshot was captured with `iptables-save -c`, the parser preserves packet/byte counts on chains and rules. The report can reference these to identify active vs. dead rules.

## Troubleshooting

| Problem | Solution |
|---|---|
| `Error: File not found` | Check the path; use an absolute path |
| `Error: iptables_parser.py not found` | Re-run the install step |
| `Error: python3 not on PATH` | Install Python 3.10+: `sudo apt install python3` |
| `Failed to parse as iptables-save` | Ensure the file is from `iptables-save`, not `iptables -L` |
| `/iptables-explain` not in autocomplete | Confirm `~/.claude/skills/iptables-explain/skill.md` exists; restart Claude Code |
| Report says "Framework: unknown" | File header may be truncated; first line must be the `# Generated by iptables-save` comment |

## Using the Parser Standalone

`iptables_parser.py` can also be used independently of the Claude skill:

```bash
# Parse to JSON (stdout)
python3 iptables_parser.py snapshot.txt

# Parse IPv6 snapshot
python3 iptables_parser.py snapshot-ip6.txt --family ipv6

# Pretty-print with 4-space indentation
python3 iptables_parser.py snapshot.txt --indent 4

# Pipe into jq
python3 iptables_parser.py snapshot.txt | jq '.diagnostics.drop_policy_chains'
```

The JSON output is fully documented in `docs/overview.md`.
